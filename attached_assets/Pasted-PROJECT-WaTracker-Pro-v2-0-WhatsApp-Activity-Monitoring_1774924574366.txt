IMPORTANT — READ BEFORE YOU START:
This is a LARGE project. Build it INCREMENTALLY using the phased plan below. Complete Phase 1 fully, test it, then move to Phase 2, and so on. Create a checkpoint after each phase. Do NOT attempt to build everything in a single pass. At each phase, ask yourself: "Am I adding any functionality that wasn't explicitly requested in this phase?" — if yes, stop and stay on track.

SECTION 1: TECH STACK & ENVIRONMENT
═══════════════════════════════════════════════════════════════════

Frontend:
- Expo SDK 54 + React Native + TypeScript
- Expo Router (file-based routing) for all navigation
- React Native 0.81, React 19.1

Backend:
- Express 5 + TypeScript + Drizzle ORM + PostgreSQL
- Already running on port 3001 — do NOT rebuild, modify, or restart the backend
- Connect to all API endpoints with credentials: 'include' on every fetch/axios call
- Use getApiUrl() from @/lib/query-client for all API base URLs — never hardcode hostnames or localhost

State Management:
- React Query (@tanstack/react-query) for all server state (fetching, caching, mutations, optimistic updates)
- React Context for auth state (AuthProvider) and theme state (ThemeProvider)
- AsyncStorage for persisting: theme preference, onboarding completion flag, DND settings, biometric lock preference, favorite contacts list

Navigation:
- NativeTabs from expo-router with isLiquidGlassAvailable() check — use liquid glass tab bar on iOS 26+, fallback to classic Tabs with BlurView on older iOS and Android
- 5 bottom tabs + stack screens pushed on top

Styling:
- React Native StyleSheet ONLY — no styled-components, no NativeWind, no Tailwind
- ALL color values come from constants/colors.ts via useColors() hook
- Never write hardcoded hex values inside component files
- Font: Inter (use Inter_400Regular, Inter_500Medium, Inter_600SemiBold, Inter_700Bold via expo-font — Inter is pre-installed)

Key Libraries (install these):
- react-native-reanimated (v4) — for all animations
- @tanstack/react-query — server state
- @react-native-async-storage/async-storage — persistence
- expo-haptics — haptic feedback
- expo-sharing — file sharing/export
- expo-local-authentication — biometric lock
- expo-notifications — push notifications
- expo-secure-store — store sensitive tokens
- @expo/vector-icons (Ionicons, Feather, MaterialCommunityIcons) — icons only, NO emojis anywhere
- react-native-gesture-handler — swipe actions
- react-native-safe-area-context — safe areas
- expo-blur — BlurView for tab bar fallback
- expo-linear-gradient — gradient backgrounds
- date-fns — relative time formatting
- react-native-toast-message — toast notifications

Do NOT install: any charting library (build all charts with pure React Native Views), any CSS-in-JS library, any emoji package

SECTION 2: BACKEND API REFERENCE (EXISTING — DO NOT MODIFY)
═══════════════════════════════════════════════════════════════════

Auth:
POST /api/auth/register — body: { username, email, password } → returns user object
POST /api/auth/login — body: { email, password } → sets httpOnly JWT cookie
POST /api/auth/logout — clears cookie
GET  /api/auth/me — returns current user or 401

Contacts:
GET    /api/contacts — returns all tracked contacts
POST   /api/contacts — body: { name, phoneNumber }
PUT    /api/contacts/:id — body: { name, notes, alertEnabled }
DELETE /api/contacts/:id

Sessions & Stats:
GET  /api/contacts/:id/sessions?from=&to= — activity sessions with time ranges
GET  /api/contacts/:id/stats?range=today|week|month — aggregated stats
GET  /api/contacts/:id/hourly — 24-hour activity breakdown
POST /api/contacts/:id/status — body: { status: 'online'|'offline' }

Chat & Media:
GET /api/conversations — all tracked conversations
GET /api/messages/:contactId?page=&limit= — paginated messages
GET /api/view-once — recovered view-once media items

Notifications:
GET    /api/notifications?filter=&page= — paginated, filterable
POST   /api/notifications/mark-read/:id
POST   /api/notifications/mark-all-read
DELETE /api/notifications/clear

Reports:
GET /api/reports/:contactId?range= — full activity report
GET /api/reports/:contactId/export?format=csv — export as CSV file

Subscription & Settings:
GET  /api/subscription/plans — available plans
GET  /api/subscription/current — user's current plan
POST /api/subscription/upgrade — body: { planId }
GET  /api/settings — user settings
PUT  /api/settings — update settings
POST /api/settings/dnd — body: { startTime, endTime, label }
DELETE /api/settings/dnd/:id

NEW API ENDPOINTS (add these to the existing backend pattern — frontend should call them even if backend returns 404 for now, with graceful fallback):
GET    /api/contacts/favorites — returns favorite contacts
POST   /api/contacts/:id/favorite — toggle favorite
GET    /api/contacts/:id/patterns — AI-predicted activity patterns
GET    /api/contacts/groups — contact groups
POST   /api/contacts/groups — body: { name, contactIds }
PUT    /api/contacts/groups/:id — update group
DELETE /api/contacts/groups/:id
GET    /api/activity/family-summary — aggregated family dashboard data
GET    /api/activity/comparisons?contactIds=1,2,3 — compare multiple contacts
POST   /api/alerts/keyword — body: { keyword, severity }
GET    /api/alerts/keywords — list keyword alerts
DELETE /api/alerts/keyword/:id
GET    /api/geofence/zones — geofence zones
POST   /api/geofence/zones — body: { name, lat, lng, radius }
GET    /api/activity/timeline?date= — full day timeline for all contacts

SECTION 8: CROSS-CUTTING REQUIREMENTS
═══════════════════════════════════════════════════════════════════

Safe Areas:
- useSafeAreaInsets() on EVERY screen — no hardcoded top/bottom padding anywhere
- Web platform fallback: top inset 67px, tab bar height 84px (50 + 34 bottom safe area)

Loading States:
- Every list screen must have: skeleton shimmer loading state, error state with retry button, pull-to-refresh
- Use React Query's isLoading, isError, refetch for all states
- Skeleton components match the exact layout of the real content

Haptic Feedback (expo-haptics):
- Impact Medium: add contact success, mark notification read, favorite toggle, tab switch
- Impact Heavy: destructive action confirmation (delete contact, clear notifications, delete account)
- Selection: filter chip tap, toggle switch

Destructive Actions:
- ALL destructive actions use Alert.alert with a red "Delete" styled button + "Cancel" button
- Delete account uses 3-step escalating confirmation (described in Settings)

Timestamps:
- All timestamps are relative: "just now", "3 min ago", "1h ago", "Yesterday 22:15", "Mar 28 14:30"
- Auto-update every 60 seconds via useRelativeTime hook (useEffect + setInterval that triggers re-render)
- Use date-fns for formatting

Online Pulsing Dot:
- Reusable PulsingDot component
- react-native-reanimated: useSharedValue + withRepeat + withSequence + withTiming
- Scale: 1 → 1.4 → 1, duration 1200ms total, easing: Easing.inOut(Easing.ease)
- Green (#25D366) fill with slight green shadow/glow

Error Handling:
- Custom ErrorBoundary component wrapping the entire app
- On error: shows error message + "Restart App" button using reloadAppAsync from expo
- Per-screen: error state with icon + message + "Try Again" button that calls refetch()
- API errors: toast notification with error message

Accessibility:
- All touchable elements have accessibilityLabel and accessibilityRole
- Minimum touch target: 44x44
- Color contrast ratios meet WCAG AA standards
- Screen reader support for all status indicators ("John is currently online")

Performance:
- FlatList with scrollEnabled={data.length > 0} to prevent empty scroll bounce
- FlatList with getItemLayout where possible for fixed-height rows
- React.memo on all list item components
- useCallback on all event handlers passed to lists
- Lazy load heavy screens using React.lazy + Suspense
- Image caching for avatars and media thumbnails

Biometric Lock (NEW):
- When enabled in settings: uses expo-local-authentication
- Triggers when app returns from background after configured delay
- Shows full-screen overlay with lock icon + "Unlock with Face ID" button
- Fallback: "Use Passcode" option
- On failure: stays locked, shows error message
- Persists lock preference + timeout to AsyncStorage

Push Notifications (NEW):
- expo-notifications for registration and handling
- NotificationProvider in root layout handles permission request on first launch
- Notification tap navigates to relevant screen (contact detail, chat, notifications tab)
- Badge count synced with unread notifications count

Network Status (NEW):
- Detect offline state using NetInfo or equivalent
- Show persistent banner at top: "No internet connection" in red
- Queue failed mutations and retry when back online (React Query offline support)
- Show cached data with "Last updated X minutes ago" indicator
