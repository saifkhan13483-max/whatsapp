Build a WhatsApp Web linking system for my app using the whatsapp-web.js library. The system must support two connection methods:
Method 1 — QR Code Scanning:
Generate a QR code image using the qrcode npm package from the qr event emitted by the WhatsApp client. Display it in the browser as a <img> tag with the base64 data URL. Refresh it automatically every 4 seconds by polling GET /api/whatsapp/qr.
Method 2 — Pairing Code (phone number):
After the WhatsApp client is initialized and the first qr event fires (which means the client is ready to accept a pairing code), call client.requestPairingCode(phoneNumber) where phoneNumber is the user's number in international format without the + (e.g. 12025551234). Display the returned 8-character pairing code (e.g. ABCD-1234) prominently on screen. The user enters this code on their phone at WhatsApp → Linked Devices → Link with phone number → Enter code.
Backend requirements (waClient.js / server.js):
Use whatsapp-web.js with LocalAuth strategy to persist the session to ./.whatsapp-session

Expose these REST API endpoints:
GET /api/whatsapp/status → { ready, initializing, hasQR, pairingCode, phoneNumber, authorizedNumber, messageCount }
GET /api/whatsapp/qr → { qr: "<base64 data URL>" }
POST /api/whatsapp/start → starts the bridge (fire-and-forget, returns immediately)
POST /api/whatsapp/stop → destroys the client
POST /api/whatsapp/pairing-code with body { number: "12025551234" } → calls client.requestPairingCode(number) and returns { code: "ABCD-1234" }
POST /api/whatsapp/authorize with body { number: "12025551234" | null } → sets the authorized sender number (null = allow all)
GET /api/whatsapp/messages → returns last 50 messages log
